DATA FLOW ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Simulador (números aleatorios)
    ↓
    ├─ POST :4041/iot/json?i=air-sensor-madrid&k=iot-agent-api-key-prod
    │  └─ {"pm10": 65.4, "pm25": 28.5, "no2": 89.5, ...}
    ↓
IoT Agent JSON (transforma a NGSI-LD)
    ├─ object_id "pm10" → name "PM10"
    ├─ object_id "laeq" → name "LAeq"
    └─ entity_type "AirQualityObserved"
    ↓
Orion-LD :1026 (Context Broker - datos ACTUALES)
    ├─ Crea entidad: urn:ngsi-ld:AirQualityObserved:Madrid:Centro
    ├─ Atributos: PM10, PM2_5, NO2, O3, temperature, relativeHumidity
    └─ Timestamp: dateObserved
    ↓ [SUSCRIPCIÓN AUTOMÁTICA]
    │
QuantumLeap :8668 (Time Series API)
    ├─ Recibe notificaciones de cambios
    └─ Guarda en TimescaleDB
    ↓
Frontend :3000 (app_fiware.js)
    ├─ fetch Orion → datos ACTUALES (PM10: 65.4)
    ├─ fetch QuantumLeap → históricos [65.4, 63.8, 67.2, ...]
    └─ Renderiza tarjetas con ambas fuentes
